# OpenClaw Development Rules

- **Anti-Hallucination Mandate**: When building or modifying OpenClaw features (e.g., wizards, configuration files, core behaviors), you MUST NOT hallucinate capabilities, config keys, or file formats. If you are uncertain or lack complete information, you MUST use your web reading tools to search and read the official documentation starting from `https://docs.openclaw.ai/` before implementing anything.

## OpenClaw Path Architecture

OpenClaw reads its workspace from `~/.openclaw/workspace/`. In this project's Docker container:

```
~/.openclaw/workspace/       (symlink)
       ↕
/home/node/what-watch-bot/   (populated via: COPY workspace/ /home/node/what-watch-bot/)
       ↕
workspace/                   (in the repo — this is the source of truth)
```

**Skills** are loaded from `<workspace>/skills/`. In this project:

- Repo path: `workspace/skills/what-watch-bot/`
- Container path: `/home/node/what-watch-bot/skills/what-watch-bot/`
- OpenClaw sees it as: `<workspace>/skills/what-watch-bot/`

## Workspace-Level Files (live in `workspace/` root)

These are native OpenClaw template files. NEVER invent new workspace-level files without verifying against the OpenClaw standard at `https://docs.openclaw.ai/reference/templates/`:

| File                   | Purpose                                                                                                         | Generated by OpenClaw if missing? |
| ---------------------- | --------------------------------------------------------------------------------------------------------------- | --------------------------------- |
| `AGENTS.md`            | Session loop controller. On first run calls `BOOTSTRAP.md`. Each session reads `SOUL.md`, `USER.md`, `memory/`. | ✅ Yes                            |
| `BOOTSTRAP.md`         | First-run setup wizard. Creates `IDENTITY.md` and `USER.md`.                                                    | ✅ Yes                            |
| `SOUL.md`              | Agent identity: Core Truths, Boundaries, Vibe, Continuity.                                                      | ✅ Yes (default)                  |
| `IDENTITY.md`          | Name, Creature, Vibe, Emoji, Avatar path.                                                                       | ✅ Yes (default)                  |
| `USER.md`              | Information about the human user.                                                                               | ✅ Yes                            |
| `HEARTBEAT.md`         | Proactive background tasks. Keep empty/comments-only to skip heartbeat API calls.                               | ✅ Yes                            |
| `TOOLS.md`             | Local Notes: environment-specific configs (SSH hosts, voices, device names).                                    | ✅ Yes                            |
| `BOOT.md`              | Internal hooks executed at system boot (`hooks.internal.enabled`).                                              | ✅ Yes                            |
| `MEMORY.md`            | Long-term curated memory.                                                                                       | Created on first write            |
| `memory/YYYY-MM-DD.md` | Short-term daily raw logs.                                                                                      | Created on first write            |

> [!IMPORTANT]
> Files that OpenClaw can auto-generate (`SOUL.md`, `IDENTITY.md`, etc.) will be generated with **default content** if not present in the workspace. Our custom versions in `workspace/` override the defaults by being present at container start.

## Skill-Level Files (live in `workspace/skills/what-watch-bot/`)

| File       | Purpose                                                                          |
| ---------- | -------------------------------------------------------------------------------- |
| `SKILL.md` | The skill manifest: slash command mapping, tool documentation, bot instructions. |
| `scripts/` | Python entrypoints called by OpenClaw actions.                                   |
| `src/`     | Application logic (api/, utils/, config.py, database.py).                        |
| `tests/`   | Dev-only. Excluded from Docker image via `.dockerignore`.                        |
